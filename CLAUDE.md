# CLAUDE.md

「若者にギフトを広めるAIフライヤー」活動の制作基盤プロジェクト。
食品ギフト全般の販促フライヤーを pptxgenjs で生成する。

## このプロジェクトでやること

- 商品フォルダ（`products/<商品名>/`）の画像と商品情報からPPTXフライヤーを生成
- キャッチコピー・商品説明・訴求ポイントの文案作成
- 案数はテーマ次第で複数パターン提案。各案は構造から別物にする

## 読むべきファイル

| やりたいこと | 読むファイル |
|---|---|
| **作業の進め方・品質基準（全作業共通の行動規範）** | `.claude/skills/context-framework.md` |
| 活動の目的・ターゲット・トーン・実績 | `.claude/skills/gift-info.md` |
| フライヤーを作る | `.claude/skills/flyer/flyer-prompt.md` |
| **制作ノウハウ（技術Tips・QA手順・手動編集保護）** | `.claude/skills/flyer/flyer-knowhow.md` |

商品固有の情報は `products/<商品名>/product-info.md`（自由記述）を読む。

## 出力の基本ルール

- 生成した .pptx とスクリプトは `flyers/<商品名>/` に保存する
- 生成後は必ず `export-slides.ps1` でPNG書き出し→目視QA→修正を収束するまで回してから見せる
- JPG納品・承認処理は、ユーザーが完成版を明示的に承認したときだけ行う
- **`index.html` のカード追加・JPG変換は、ユーザーが完成版を明示的に承認したときだけ行う（承認前に書き換えない）**。試作段階のデザインはチャット上のプレビュー画像で確認する
- 日本語メイン、ターゲットは10代後半〜20代＋ギフト購入者
- **参考資料（`references/`）だけに頼らない。** テーマは商品写真・商品情報の物語から複数発想し、既存の意匠に合いそうな時だけ参考にする（`flyer-prompt.md`「根本原則」）。同じ骨格の使い回し（情報カード＋写真＋箇条書きの繰り返し）に陥っていないか、複数案を並べて自己点検する

## 手動編集保護（例外なし・最重要）

**一度ユーザーに渡した pptx は手動編集されている前提で扱う。**

- **pptx を再生成・上書きする前に、必ず `scripts/check-flyer-hash.ps1` で照合する。** 不一致（=手動編集の可能性）なら絶対に上書きせず、ユーザーに確認する
- **これはコードでも強制されている**：生成スクリプトは `flyers/_lib/pptx-helpers.js` の `safeWriteFile()` 経由でしか書き込まず、ハッシュ不一致なら Claude の判断に関係なく書き込みを拒否する（`flyer-knowhow.md` §7）。ただし「承認記録なし」の場合はコードでは判断できないので、上記の事前照合とユーザーへの確認は引き続き必須
- **PowerPoint が開いていて再生成できない（EBUSY）時は、理由を決めつけて無断でプロセスを閉じるな。必ずユーザーに聞く。** 「PowerPointが開いていて再生成できません。編集中でしたら保存してから教えてください」と聞くだけでいい
- ユーザーが承認したら `scripts/approve-flyer.ps1` でハッシュ記録 + git commit でスナップショット化する
- 過去に姉妹プロジェクト（hotdog-stall）でこれを怠り、ユーザーの手動編集を握りつぶした事故がある（詳細は `.claude/skills/flyer/flyer-knowhow.md` §7）。例外なし。
