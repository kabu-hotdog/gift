# flyer-prompt.md — pptxgenjs フライヤー生成スキル（商品非依存）

> 前提：`.claude/skills/gift-info.md` と `products/<商品名>/product-info.md` を先に読む。
> 技術知見・QA手順・手動編集保護は `.claude/skills/flyer/flyer-knowhow.md` を参照。
> 出力：pptxgenjs で生成した .pptx。**1つの商品フォルダから複数パターン作る（案数はテーマ次第。1ファイルに複数スライドでまとめてよい）**。保存先 `flyers/<商品名>/`

---

## 根本原則：凡庸を殺せ

**以下はすべて禁止。やった時点でリセットして作り直す。**

- 「上に写真／中にキャッチ／下に情報」の量産型三段構成
- メイン画像＋サムネイル並列という汎用レイアウト
- 白または薄いベージュの地色に普通サイズのテキストを並べただけの構成
- 複数案が「同じ構造の色違い」になること
- 商品画像を小さなアクセントとして添える扱い
- 情報をただ並べただけのレイアウト

**「見た人が一瞬で止まる」強度を最低ラインとする。**
縦書き・斜め・断ち落とし・特大タイポ・全面色面・透かし大文字・コラージュ・
重ね・はみ出し・回転・額装・タイル・帯状反復——迷ったら「やりすぎ」の方を選べ。
提供画像は「主役/脇役」の発想を捨て、**構図を作る部材**として扱う。余白も造形として効かせる。
各案に必ず1つ「大胆な仕掛け」を入れる。

---

## 制作フロー（この順番を崩さない）

1. **素材スキャン**：`products/<商品名>/images/` と `product-info.md` を読む
2. **しずる写真の見極め**（下記）
3. **テーマ発想**：写真・商品が持つ物語から逆算する（`flyer-knowhow.md` §8 のフレームワーク）。
   固定テンプレ・スタイル軸は使わない。新ジャンルならリサーチしてそのジャンルの記号を3つ以上入れる
4. **コピー作成**（案ごとに書き分け。下記）
5. **生成**：pptxgenjs（実装規約は `flyer-knowhow.md` §6）
6. **QAループ**：`export-slides.ps1` でPNG書き出し→目視→修正、収束するまで（`flyer-knowhow.md` §1）
7. **提示**：仕上がってから。承認されたら `scripts/approve-flyer.ps1` + git commit（`flyer-knowhow.md` §7）

### 素材スキャン

```js
const fs = require('fs');
const path = require('path');

function scanImages(dir) {
  if (!fs.existsSync(dir)) return [];
  return fs.readdirSync(dir)
    .filter(f => /\.(png|jpg|jpeg|webp|gif)$/i.test(f))
    .map(f => path.join(dir, f));
}
```

- 画像は**ファイル名で解決**する（インデックス参照禁止。`flyer-knowhow.md` §7.7）
- 各画像について「内容・しずる候補か・透過か白背景か」をメモしてから設計に入る
- 透過PNGはアルファbboxを実測してから使う（透明余白で縮む事故。`flyer-knowhow.md` §5）

## しずる写真の見極めと運用（重要）

- 提供画像の中から「**最も食欲をそそる一枚**」を特定する。
  目安＝商品が複数並ぶ／自然光・湯気・水滴・断面・具材やフルーツが添う／食卓やスタイリングがある
- しずる写真は**大多数の案でメイン級**に使う。ただし見せ方を案ごとに変える（額装・大きく断ち落とし・円形抜き・帯状に沈める）
- 全案で大きく使う必要はない。ポップ/グラフィック寄りの案では小さく or 不使用が正解になることもある（無理に入れず、その案の世界観を優先）
- **透過PNGの単体カット**は地色に直接重ねられる。**白背景の写真**は「金枠で額装」「カードに乗せる」「地色を半透明で被せて沈める」で地色に馴染ませる
- 写真は非破壊が原則：円形cover-cropで商品が切れるくらいなら、円形の台紙の上に透過PNGを重ねる（`flyer-knowhow.md` §7.6）

## ロゴと手書き文字

- 商品にロゴ/マークがあれば、図形（OVAL+テキスト等）で再現して**ブランドの軸として全案にあしらう**（色は各案のトーンに合わせる。高級感なら金など）
- 手書き文字は活字とのコントラストで使う。効かせどころを絞る（商品名・サイン・しずる文など）。1案で活字2〜3種＋装飾フォント程度に
- 使えるフォントは環境依存。**必ず `flyer-knowhow.md` §7.5 でインストール状況を確認してから使う**

## コピー（各案で書き分け。同じコピーを2案以上で使い回さない）

- 型は決めない。**その案のテーマの文法で書く**（テーマが号外なら見出し語、手紙なら語りかけ、など）
- 情報量は極限まで削る。書きたいことはコピー1本に集約し、必須情報（商品名・価格・買える場所or期間）だけ独立させて大きく置く
- しずる文・オノマトペ（とろ〜り・ほろっ・じゅわっ・ぱりっ等）は**独立したビジュアル要素**として斜めに大きく置く（40〜60pt目安）。引き出し線でつなぐと台無し。ブロック文章に埋めるな
- 価格・訴求ポイントは恥ずかしいくらい大きく（本文の3倍以上）
- 文案はコピペしてすぐ使える完成形でも提示する

---

## 配色（商品から導く・固定しない）

- 商品写真から主役色を1つ抽出し、それを軸に各案の質感を作る
- **3色まで**＋比率 **70:25:5**。食品は暖色を必ず一点、冷/清涼系は寒色も併用
- 詳細・実績パレットは `flyer-knowhow.md` §3

## pptxgenjs 実装ルール（要点。全表は `flyer-knowhow.md` §6）

```js
const PptxGenJS = require('pptxgenjs');
const pptx = new PptxGenJS();
pptx.defineLayout({ name: 'A4', width: 8.27, height: 11.69 });
pptx.layout = 'A4';
```

| ルール | 詳細 |
|---|---|
| `hex` に `#` を付けない・8桁hex禁止 | 透明度は `transparency` / `opacity` で |
| 画像は実寸アスペクト比を渡す `addCover()` 方式 | pptxgenjsのcover/containは実画像サイズを読まないバグあり（§6） |
| 円形抜きは正方形領域 `cover` → `rounding:true` | 非正方形だと楕円になる |
| `shadow` 等オブジェクトは毎回 `{}` で新規生成 | 共有参照バグ防止 |
| `shadow: { type: 'inner' }` 使用禁止 | 壊れたXMLでPowerPointが開けなくなる |
| 断ち落としは座標負値 or W/H超でよい | 意図的なブリードはOK。重要文字の見切れはNG |
| 明るい地に白文字禁止 / 暗い地に黒文字禁止 | コントラスト必須確認 |
| 縦書きは `vert:'eaVert'` だが促音で蛇行する | 1文字ずつ個別ボックスの `vText()` 方式を推奨（§6） |

## 保存・出力・承認

```js
pptx.writeFile({ fileName: 'flyers/<商品名>/[内容が分かるファイル名].pptx' });
```

- pptx はユーザーに渡す**前**なら何度でも上書き生成してよい
- **一度ユーザーに見せた pptx を再生成する前は、必ず `scripts/check-flyer-hash.ps1` で照合**（`flyer-knowhow.md` §7。例外なし）
- ユーザーが承認したら：`scripts/approve-flyer.ps1` でハッシュ記録 → git commit → JPG書き出し（`export-slides.ps1 -Format JPG`）
- 納品時案内：pptxgenjs はフォントを埋め込まない。配布前に PowerPoint「ファイル→オプション→保存→ファイルにフォントを埋め込む」を有効化するよう必ず添える

## QAチェックリスト（生成後、毎サイクル自己確認）

```
□ しずる写真が大多数の案でメイン級に扱われているか（ポップ/グラフィック案の例外を除く）
□ 案どうしの構図が互いに完全に別物か（色違い・レイアウト違いだけ = NG）
□ 各案の「大胆な仕掛け」が機能しているか
□ 商品名・価格が記載され、価格はヒーロー扱いか（小さく添えるだけ = NG）
□ しずる文 or オノマトペが独立した大きなテキスト要素になっているか
□ 画像の縦横比が崩れていないか（addCover方式を使ったか）
□ テキストの意図しない折返し・はみ出しがないか（wrap:false + 十分な幅）
□ 要素の重なり・衝突がないか（ラベル同士のy座標、写真とカード）
□ 縦書きの列折り・蛇行がないか
□ 地色×文字色のコントラストが確保されているか
□ 顔・商品にテキストを重ねていないか
□ hex に # が混入していないか / inner shadow を使っていないか
□ 「上に写真／中にキャッチ／下に情報」になっていないか
□ 縮小表示で一番伝えたい情報が最初に目に入るか
```
